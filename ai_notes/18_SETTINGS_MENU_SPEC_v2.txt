SECTION: Autocorrect (per input mode) — beta 0.0.0.11 UI; behavior Block 5
--------------------------------------------------------------------------------
  [Toggle] Heatmap autocorrect for tap typing — default OFF
           OFF: HeliBoard Text correction / autocorrect unchanged for tapped words.
           ON:  Heatmap System-2 may replace committed tap words when implemented.
           Learning from taps still runs when master learning is ON.

  [Toggle] Heatmap autocorrect for swipe typing — default OFF
           OFF: HeliBoard glide autocorrect unchanged for swiped words.
           ON:  Heatmap System-2 may replace committed swipe words when implemented.
           Example: tap OFF + swipe ON → learn from both; only swipe path uses Heatmap AC.

  HeliBoard global autocorrect / suggestion settings are never overridden by these
  toggles when OFF for that mode. When ON but Block 5 not shipped, UI shows armed but
  runtime still uses HeliBoard (HeatmapAutocorrectPolicy_v1 implementation gate).

  [Button] Clear all learning data (heatmap, weights, typo list stats)
  [Button] Reset key geometry to default (tap / swipe separate sub-confirm)
