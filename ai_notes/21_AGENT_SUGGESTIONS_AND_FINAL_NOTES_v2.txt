SUGGESTIONS (agent recommendations)
--------------------------------------------------------------------------------

S1. Word slot ID
    Every composing word gets a monotonic slotId for the session so "tap twice" and
    swipe-delete-tap rules never bleed into the next word.

S2. Confidence gate before commit
    On final swipe lift, if top word beats second by < margin, show strip instead of
    auto-commit — reduces wrong STARTING-style commits while learning is immature.

S3. Structured debug export for AI review
    Settings toggle exports last N sessions as JSON (no full paragraphs unless user opts in):
    {swipePoints, runnerUp, outcome, corrections} for agent-assisted tuning in chat.

S4. Cap prefix branching
    Hard limit active trie branches during "s*" phase; rely on next letters to narrow.
    Prevents RAM spikes on first letter.

S5. Curve template library per user
    After ~50 corrections involving curves, cluster curve hashes → preferred letter.
    This is where "loops" get good without manual coding every shape.

S6. Common Typo List vs dictionary add
    When tap-after-swipe finds no match, offer "Add [word] to dictionary?" once — do not
    auto-add (prevents polluting user dict with typos).

S7. Testing phrases
    Maintain assets/test_swipes.txt with polylines for PAUSE, PHASE, PLASTIC, STARTING
    for regression on each phone test pass.

S8. Rename sweep before code
    When coding starts, grep "naughty" and replace with CommonTypo in user strings only;
    keep internal names consistent in one pass.
